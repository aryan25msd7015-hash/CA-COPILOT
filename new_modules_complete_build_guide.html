Advanced Compliance Calendar — client-aware, risk-aware

Not just a list of dates. Every deadline shows which specific clients are at risk, which haven't sent data, which have a history of lateness, and which already have open notices for the same period. CAs see the full picture in one grid.

What it looks like vs a basic calendar

Step 1 — Schema upgrade
Client-deadline mapping table
Every deadline must be linked to the specific clients it applies to — not all filings apply to all clients (e.g., AOC-4 only applies to companies, not proprietorships). Add a mapping table and applicability flags.
CREATE TABLE deadline_client_map ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), org_id UUID REFERENCES organizations(id), client_id UUID REFERENCES clients(id), filing_type TEXT, -- GSTR1,GSTR3B,TDS_24Q,ROC_AOC4,ADV_TAX,MSME_AUDIT period TEXT, -- '2024-10' deadline DATE, -- DATA STATUS data_received BOOLEAN DEFAULT FALSE, data_received_at TIMESTAMPTZ, data_source TEXT, -- 'whatsapp','upload','erp' -- FILING STATUS status TEXT DEFAULT 'pending', -- pending,filed,missed filed_at TIMESTAMPTZ, -- RISK SIGNALS late_count_last_12m INT DEFAULT 0, -- computed nightly has_open_notice BOOLEAN DEFAULT FALSE,-- computed nightly risk_score NUMERIC(3,1) DEFAULT 5.0, -- 0-10, computed -- WA REMINDERS reminders_sent INT DEFAULT 0, last_reminder_at TIMESTAMPTZ );
Step 2 — Nightly risk scorer
Compute per-client risk score for each upcoming deadline
Run nightly via Celery Beat. Each client-deadline pair gets a 0–10 risk score based on 5 signals. The calendar sorts by this score — highest risk floats to top.
@celery_app.task def score_deadline_risks(): """Runs nightly at 8pm — before morning reminders""" upcoming = db.query(DeadlineClientMap).filter( DeadlineClientMap.deadline >= date.today(), DeadlineClientMap.deadline <= date.today()+timedelta(days=14), DeadlineClientMap.status == 'pending' ).all() for row in upcoming: score = 0.0 days_left = (row.deadline - date.today()).days # Signal 1: Days remaining (closer = higher risk) if days_left <= 2: score += 3.0 elif days_left <= 5: score += 2.0 elif days_left <= 10:score += 1.0 # Signal 2: Data not yet received if not row.data_received: score += 2.5 # Signal 3: Client's late history for this filing type if row.late_count_last_12m >= 3: score += 2.0 elif row.late_count_last_12m >= 1: score += 1.0 # Signal 4: Open notice for same filing type if row.has_open_notice: score += 1.5 # Signal 5: Health score below 50 client = db.query(Client).get(row.client_id) if client.health_score < 50: score += 1.0 row.risk_score = min(score, 10.0) db.commit()
Step 3 — Applicability engine
Auto-tag which filings apply to which clients
CLIENT_FILING_MATRIX = { 'proprietorship':['GSTR1','GSTR3B','ITR','TDS_26Q'], 'partnership': ['GSTR1','GSTR3B','ITR','TDS_26Q','FORM_3CB'], 'pvt_ltd': ['GSTR1','GSTR3B','TDS_24Q','TDS_26Q', 'ROC_AOC4','ROC_MGT7','ADV_TAX','FORM_3CA'], 'llp': ['GSTR1','GSTR3B','TDS_26Q','ROC_LLP8', 'ROC_LLP11','ADV_TAX'], 'trust': ['ITR_7','FCRA_IF_APPLICABLE'], } def seed_client_deadlines(client_id: str, period: str): client = db.query(Client).get(client_id) applicable = CLIENT_FILING_MATRIX.get(client.entity_type, []) for filing in applicable: deadline = compute_deadline(filing, period) if deadline: row = DeadlineClientMap( org_id=client.org_id, client_id=client_id, filing_type=filing, period=period, deadline=deadline, late_count_last_12m=get_late_count(client_id,filing)) db.merge(row) db.commit()
Step 4 — Calendar API endpoint
Single endpoint powering the entire calendar view
@router.get("/calendar/overview") async def calendar_overview( days_ahead: int = 30, req: Request = None): rows = db.execute(""" SELECT dcm.filing_type, dcm.deadline, dcm.period, COUNT(*) FILTER(WHERE dcm.status='pending') AS total_pending, COUNT(*) FILTER(WHERE dcm.risk_score >= 7) AS high_risk_count, COUNT(*) FILTER(WHERE dcm.data_received=FALSE AND dcm.status='pending') AS data_missing, COUNT(*) FILTER(WHERE dcm.late_count_last_12m >= 2) AS chronic_late, COUNT(*) FILTER(WHERE dcm.has_open_notice=TRUE) AS has_notice, MAX(dcm.risk_score) AS max_risk, -- Aggregate deadline-level status CASE WHEN MAX(dcm.risk_score)>=7 THEN 'urgent' WHEN MAX(dcm.risk_score)>=4 THEN 'review' ELSE 'on_track' END AS priority FROM deadline_client_map dcm WHERE dcm.org_id = :oid AND dcm.deadline BETWEEN NOW() AND NOW()+(:days||' days')::interval GROUP BY dcm.filing_type, dcm.deadline, dcm.period ORDER BY dcm.deadline ASC, max_risk DESC """, dict(oid=req.state.org_id, days=days_ahead)).fetchall() return [dict(r) for r in rows] @router.get("/calendar/{filing_type}/{period}/clients") async def deadline_clients( filing_type: str, period: str, req: Request): """Drill-down: show all clients for a specific deadline""" rows = db.execute(""" SELECT c.name, c.gstin, c.health_score, dcm.risk_score, dcm.data_received, dcm.data_source, dcm.late_count_last_12m, dcm.has_open_notice, dcm.reminders_sent, dcm.status FROM deadline_client_map dcm JOIN clients c ON c.id = dcm.client_id WHERE dcm.org_id = :oid AND dcm.filing_type = :ft AND dcm.period = :p ORDER BY dcm.risk_score DESC """, dict(oid=req.state.org_id, ft=filing_type, p=period)).fetchall() return [dict(r) for r in rows]
Step 5 — WhatsApp integration for the calendar
Bulk send reminders from the calendar view with one click
CA selects a deadline row, clicks "Remind all pending" — system sends WhatsApp reminders only to clients with data_received=False and reminders_sent less than 3. No manual selecting of individual clients.
@router.post("/calendar/{filing_type}/{period}/bulk-remind") async def bulk_remind(filing_type: str, period: str, req: Request): if req.state.role not in ('partner','manager'): raise HTTPException(403) pending = db.query(DeadlineClientMap).filter( DeadlineClientMap.org_id == req.state.org_id, DeadlineClientMap.filing_type == filing_type, DeadlineClientMap.period == period, DeadlineClientMap.data_received == False, DeadlineClientMap.reminders_sent < 3, DeadlineClientMap.status == 'pending' ).all() sent = 0 for row in pending: client = db.query(Client).get(row.client_id) if client.whatsapp_consent_at and client.whatsapp_number: send_whatsapp_template.delay( client.whatsapp_number, 'data_request_reminder', [client.name, filing_type, row.deadline.strftime('%d %b'), filing_type]) row.reminders_sent += 1 row.last_reminder_at = datetime.now() sent += 1 db.commit() return {"reminders_sent": sent, "skipped": len(pending)-sent}

MSME Payment Audit — Section 43B(h) tracker

Automatically identifies MSME vendors, monitors 45-day payment windows in the accounts payable ledger, computes disallowance amounts, and flags violations before tax audit season. Zero manual work for the CA.

Step 1 — DB tables
MSME vendor registry
CREATE TABLE msme_vendors ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), org_id UUID REFERENCES organizations(id), client_id UUID REFERENCES clients(id), vendor_gstin TEXT, vendor_name TEXT, udyam_reg_no TEXT, -- UAM/Udyam number from certificate udyam_category TEXT, -- 'micro','small','medium' udyam_cert_doc_id UUID, -- FK to documents table registered_at DATE, verified_at TIMESTAMPTZ, -- when we confirmed validity created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE msme_payment_violations ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), org_id UUID REFERENCES organizations(id), client_id UUID REFERENCES clients(id), vendor_id UUID REFERENCES msme_vendors(id), invoice_id UUID REFERENCES transactions(id), invoice_date DATE, invoice_amount NUMERIC(15,2), due_date DATE, -- invoice_date + 45 days payment_date DATE, days_overdue INT, disallowance_amount NUMERIC(15,2), interest_amount NUMERIC(15,2), fy TEXT, -- '2024-25' status TEXT DEFAULT 'open' -- open, cleared, reported );
Step 2 — OCR Udyam certificate
Auto-extract vendor MSME classification from uploaded certificates
@celery_app.task(queue='ocr') def process_udyam_cert(document_id: str, client_id: str): doc = db.query(Document).get(document_id) # Use Azure OCR result = run_azure_ocr(doc.s3_key) text = result.content # Extract Udyam fields with regex udyam_no = re.search(r'UDYAM-[A-Z]{2}-\d{2}-\d{7}', text) category = re.search( r'(MICRO|SMALL|MEDIUM)\s+ENTERPRISE', text, re.I) name_match= re.search(r'Name of Enterprise[:\s]+(.+)', text) gstin_match=re.search(r'[0-9]{2}[A-Z]{5}[0-9]{4}[A-Z][1-9A-Z]Z[0-9A-Z]',text) if udyam_no and category: vendor = db.query(MsmeVendor).filter( MsmeVendor.org_id == doc.org_id, MsmeVendor.vendor_gstin == (gstin_match.group() if gstin_match else None) ).first() or MsmeVendor(org_id=doc.org_id, client_id=client_id) vendor.udyam_reg_no = udyam_no.group() vendor.udyam_category = category.group(1).lower() vendor.vendor_name = name_match.group(1).strip() if name_match else "" vendor.udyam_cert_doc_id = doc.id vendor.verified_at = datetime.now() db.merge(vendor); db.commit() # Now scan AP ledger for violations scan_msme_violations.delay(str(vendor.id))
Step 3 — 45-day violation scanner
Cross-reference AP ledger against MSME vendor list
@celery_app.task def scan_msme_violations(vendor_id: str): vendor = db.query(MsmeVendor).get(vendor_id) # Get all unpaid/late invoices for this MSME vendor invoices = db.query(Transaction).filter( Transaction.client_id == vendor.client_id, Transaction.vendor_gstin == vendor.vendor_gstin, Transaction.date >= date(date.today().year-1, 4, 1) # current FY ).all() for inv in invoices: due_date = inv.date + timedelta(days=45) payment = get_payment_date(inv.id) # from payment ledger if payment is None or payment > due_date: days_overdue = ((payment or date.today()) - due_date).days disallowance = inv.amount # full amount disallowed # Interest u/s 16 of MSMED Act: 3x bank rate interest = inv.amount * 0.045 * (days_overdue/365) violation = MsmePaymentViolation( org_id=vendor.org_id, client_id=vendor.client_id, vendor_id=vendor.id, invoice_id=inv.id, invoice_date=inv.date, invoice_amount=inv.amount, due_date=due_date, payment_date=payment, days_overdue=days_overdue, disallowance_amount=disallowance, interest_amount=round(interest,2), fy=get_fy(inv.date) ) db.merge(violation) db.commit() def get_fy(dt: date) -> str: y = dt.year if dt.month >= 4 else dt.year - 1 return f"{y}-{str(y+1)[-2:]}"
Step 4 — Tax audit report auto-population
Generate Clause 22 of Form 3CD from violation data
def generate_clause_22_data(client_id: str, fy: str) -> dict: """Clause 22 of Form 3CD: MSME disallowances u/s 43B(h)""" violations = db.query(MsmePaymentViolation).filter( MsmePaymentViolation.client_id == client_id, MsmePaymentViolation.fy == fy, MsmePaymentViolation.status == 'open' ).all() return { "clause": "22 - Section 43B(h)", "total_disallowance": sum(v.disallowance_amount for v in violations), "total_interest": sum(v.interest_amount for v in violations), "vendor_count": len(set(v.vendor_id for v in violations)), "violations": [{ "vendor_name": get_vendor_name(v.vendor_id), "udyam_category": get_vendor(v.vendor_id).udyam_category, "invoice_date": str(v.invoice_date), "due_date": str(v.due_date), "days_overdue": v.days_overdue, "disallowance": float(v.disallowance_amount), "interest": float(v.interest_amount), } for v in violations] }

Bank Drawing Power & Stock Statement engine

Reads raw inventory + debtor ledgers, strips non-eligible items, applies bank-specific margin formulas, and generates monthly DP statements automatically. Replaces 4–6 hours of manual work per client per month.

Step 1 — DB tables
Bank facility + DP configuration per client
CREATE TABLE bank_facilities ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), client_id UUID REFERENCES clients(id), bank_name TEXT, facility_type TEXT DEFAULT 'CC', -- CC, OD, WCDL sanctioned_limit NUMERIC(15,2), -- Bank-specific margin rules (stored as JSON) margin_rules JSONB, -- Example margin_rules: -- {"stock_margin":0.25, "debtor_margin":0.25, -- "stock_age_cutoff_days":180, -- "debtor_age_cutoff_days":90, -- "creditor_deduction":true} created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE drawing_power_statements ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), client_id UUID REFERENCES clients(id), facility_id UUID REFERENCES bank_facilities(id), period TEXT, -- '2024-10' gross_stock NUMERIC(15,2), eligible_stock NUMERIC(15,2), gross_debtors NUMERIC(15,2), eligible_debtors NUMERIC(15,2), creditors NUMERIC(15,2), drawing_power NUMERIC(15,2), doc_id UUID, -- generated PDF created_at TIMESTAMPTZ DEFAULT NOW() );
Step 2 — Inventory aging analyzer
Flag slow-moving and obsolete stock
def analyze_stock_eligibility(inventory_df, rules: dict): cutoff = rules.get('stock_age_cutoff_days', 180) margin = rules.get('stock_margin', 0.25) # Flag slow-moving: no movement in cutoff days inventory_df['last_movement'] = pd.to_datetime( inventory_df['last_movement_date']) inventory_df['days_since_movement'] = ( pd.Timestamp.today() - inventory_df['last_movement']).dt.days inventory_df['eligible'] = ( inventory_df['days_since_movement'] < cutoff) gross_stock = inventory_df['value'].sum() eligible_stock = inventory_df[inventory_df['eligible']]['value'].sum() stock_dp = eligible_stock * (1 - margin) slow_moving = inventory_df[~inventory_df['eligible']][[ 'sku','description','value','days_since_movement']] return { "gross_stock": float(gross_stock), "eligible_stock": float(eligible_stock), "stock_dp": float(stock_dp), "slow_moving_items": slow_moving.to_dict('records'), "slow_moving_value": float(slow_moving['value'].sum()) }
Step 3 — Debtor aging + 90-day predictor
ML model predicts which debtors will cross 90 days
def analyze_debtor_eligibility(debtors_df, rules: dict): cutoff = rules.get('debtor_age_cutoff_days', 90) margin = rules.get('debtor_margin', 0.25) debtors_df['invoice_date'] = pd.to_datetime( debtors_df['invoice_date']) debtors_df['age_days'] = ( pd.Timestamp.today()-debtors_df['invoice_date']).dt.days # Hard rule: already over 90 days = ineligible debtors_df['eligible'] = debtors_df['age_days'] < cutoff # Predictive flag: 60-90 days AND slow payer history debtors_df['at_risk'] = ( (debtors_df['age_days'].between(60, 89)) & (debtors_df['customer_avg_days_to_pay'] > 75) ) gross = debtors_df['outstanding'].sum() eligible = debtors_df[debtors_df['eligible']]['outstanding'].sum() debtor_dp= eligible * (1 - margin) return { "gross_debtors": float(gross), "eligible_debtors": float(eligible), "debtor_dp": float(debtor_dp), "at_risk_count": int(debtors_df['at_risk'].sum()), "ineligible_value": float( debtors_df[~debtors_df['eligible']]['outstanding'].sum()) }
Step 4 — Drawing power computation + PDF
def compute_drawing_power(client_id, facility_id, period): facility = db.query(BankFacility).get(facility_id) rules = facility.margin_rules stock_data = analyze_stock_eligibility(get_inventory(client_id), rules) debtor_data = analyze_debtor_eligibility(get_debtors(client_id), rules) creditors = get_creditors_current(client_id) # Standard bank DP formula dp = (stock_data['stock_dp'] + debtor_data['debtor_dp'] - (creditors if rules.get('creditor_deduction') else 0)) dp = min(dp, facility.sanctioned_limit) # cap at sanctioned limit stmt = DrawingPowerStatement( client_id=client_id, facility_id=facility_id, period=period, gross_stock=stock_data['gross_stock'], eligible_stock=stock_data['eligible_stock'], gross_debtors=debtor_data['gross_debtors'], eligible_debtors=debtor_data['eligible_debtors'], creditors=creditors, drawing_power=dp) db.add(stmt); db.commit() generate_dp_pdf.delay(str(stmt.id)) return dp

Lease intelligence — Ind AS 116 / IFRS 16

Reads 50-page lease agreements using NER + long-context RAG to extract all financial variables — start dates, rent-free periods, escalation clauses, termination options — and computes the full ROU asset and lease liability amortization schedule automatically.

Step 1 — Lease extraction with NER + LLM
Extract all key lease variables from PDF contracts
LEASE_EXTRACTION_PROMPT = """You are a financial accountant specializing in Ind AS 116 / IFRS 16 lease accounting. Extract ALL lease variables from the contract. Return ONLY JSON: { "commencement_date": "YYYY-MM-DD", "lease_term_months": , "base_rent_monthly": , "rent_free_period_months": , "security_deposit": , "escalation_clauses": [ {"effective_from_month": , "rate_pct": , "type": "fixed|CPI|negotiated"} ], "renewal_options": [ {"term_months": , "rent_monthly": , "notice_days": } ], "termination_option": {"earliest_month": , "penalty": }, "incremental_borrowing_rate_pct": , "lease_payments_schedule": [ {"month": , "payment": } // expand all months ] }""" def extract_lease_data(pdf_text: str) -> dict: resp = anthropic.messages.create( model="claude-sonnet-4-20250514", max_tokens=4000, system=LEASE_EXTRACTION_PROMPT, messages=[{"role":"user","content": f"Contract (extract all financial terms):\n{pdf_text[:8000]}"}]) raw = re.sub(r'```json|```','',resp.content[0].text).strip() return json.loads(raw)
Step 2 — Ind AS 116 amortization schedule
Compute full ROU asset depreciation + lease liability schedule
import numpy as np def compute_ind_as_116_schedule(lease: dict) -> list: ibr = lease['incremental_borrowing_rate_pct'] / 100 / 12 payments = [p['payment'] for p in lease['lease_payments_schedule']] n = len(payments) # PV of lease payments = initial lease liability pv_factors = [(1/(1+ibr))**(i+1) for i in range(n)] lease_liab_0 = sum(p*f for p,f in zip(payments, pv_factors)) rou_asset_0 = lease_liab_0 # simplified: no initial direct costs schedule = [] lease_liab = lease_liab_0 rou_asset = rou_asset_0 monthly_depreciation = rou_asset_0 / n for i, payment in enumerate(payments): interest_exp = lease_liab * ibr principal_red = payment - interest_exp lease_liab -= principal_red rou_asset -= monthly_depreciation schedule.append({ "month": i+1, "payment": round(payment,2), "interest_exp": round(interest_exp,2), "principal": round(principal_red,2), "lease_liability":round(max(lease_liab,0),2), "rou_asset": round(max(rou_asset,0),2), }) return schedule

Article clerk timesheet auditor

Analyzes actual output data (document uploads, task completion timestamps, email patterns) instead of trusting manual timesheet entries. Flags unbillable time, identifies bottlenecks, and computes true per-client profitability for the CA firm.

CREATE TABLE user_activity_log ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), org_id UUID REFERENCES organizations(id), user_id UUID REFERENCES users(id), client_id UUID REFERENCES clients(id), activity_type TEXT, -- upload, reconciliation_run, notice_draft, export, -- document_review, query_run, certificate_gen duration_seconds INT, -- computed from start/end events created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE timesheet_entries ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), org_id UUID REFERENCES organizations(id), user_id UUID REFERENCES users(id), client_id UUID REFERENCES clients(id), date DATE, hours_logged NUMERIC(4,2), task_description TEXT, billable BOOLEAN DEFAULT TRUE );
Step 2 — Activity classifier
Classify actual platform activities into billable task types

Step 3 — Partner profitability dashboard
Shows each partner which clients are over-serviced (actual hours far exceed billed hours), which article clerks are most efficient, and which engagements are running at a loss. Pure SQL analytics — no ML needed.
-- Partner dashboard query SELECT c.name AS client, SUM(al.duration_seconds)/3600.0 AS actual_hours, SUM(te.hours_logged) AS billed_hours, SUM(te.hours_logged) * 1500 AS revenue_at_1500, -- avg billing rate SUM(al.duration_seconds)/3600.0 * 800 AS cost_at_800,-- avg cost/hr (SUM(te.hours_logged)*1500 - SUM(al.duration_seconds)/3600.0*800) AS margin FROM user_activity_log al JOIN clients c ON c.id = al.client_id LEFT JOIN timesheet_entries te ON te.client_id = al.client_id AND te.user_id = al.user_id WHERE al.org_id = :oid AND to_char(al.created_at,'YYYY-MM') = :month GROUP BY c.name ORDER BY margin ASC; -- worst margin clients first
